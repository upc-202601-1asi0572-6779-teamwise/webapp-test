## Criterio 1: Trabaja en equipo para proporcionar liderazgo en forma conjunta

**[TU NOMBRE]**
**AV2:** Desarrollé la versión final del WebApp en Angular 21, integrando las vistas del ingeniero agrónomo con el backend .NET 10 desplegado en Render mediante un servicio HTTP tipado, conventional commits y GitFlow por bounded context. Lideré la investigación técnica del backend: leí más de 10 archivos del repositorio .NET, mapeé los 35 endpoints con curl y validé el flujo del agrónomo end-to-end con un script PowerShell automatizado, generando 4 documentos operativos (status del backend, path contractual, flujo paso a paso, plan de integración por fases) que sirvieron como referencia para alinear contratos y cerrar gaps. Coordiné con el equipo de backend la estrategia de integración, identificando limitaciones críticas (tabla `users` faltante por migración, recursos sin `id` en el body, atributos `[Authorize]` faltantes) y proponiendo workarounds accionables para la entrega final.

## Criterio 2: Crea un entorno colaborativo e inclusivo, establece metas, planifica tareas y cumple objetivos

**[TU NOMBRE]**
**AV2:** Grabé el video de 1:30 min del equipo, documentando el testimonio a cámara y mostrando el flujo end-to-end del ingeniero agrónomo (crear → aprobar → publicar → registrar intervención) con el stack completo (Angular 21, .NET 10 con DDD, firmware embedded C++). Organicé la estructura final del WebApp en 7 ramas de bounded context bajo GitFlow con conventional commits, lo que facilitó la integración modular, el code review por bounded context y el merge limpio a `main`. Establecí metas concretas para la entrega final —cablear el front al backend real, automatizar el flujo del agrónomo con un script ejecutable y documentar el estado del backend— y las cumplí dentro del plazo, dejando además un plan de integración por fases que el equipo puede iterar en sprints futuros.
